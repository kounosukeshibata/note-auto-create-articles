# プロンプト管理

## 概要

`prompts/vertex/` 配下のファイルは Gemini API に渡す生成プロンプトを管理する。  
プロンプトはアプリの記事品質を決定する核心であり、変更時は必ずこのドキュメントに記録すること。

## ファイル構成

```
prompts/vertex/
  content-generation.md   記事本文生成プロンプト（本番稼働中）
```

## 変更ルール

1. **変更前にバックアップ**: 大きな変更の場合は `content-generation-vYYYYMMDD.md` として旧バージョンを保存
2. **変更理由を記録**: このドキュメントの「変更履歴」セクションに記載
3. **検証フロー**: `/prompt-engineering` スキルの変更フローに従う
4. **git commit**: コミットメッセージに変更内容と目的を明記

## プレースホルダー仕様

| プレースホルダー | 型 | 必須 | 説明 |
|---|---|---|---|
| `{{theme}}` | string | ✓ | 記事テーマ |
| `{{article_type}}` | string | ✓ | 記事タイプ（ハウツー / レビュー / まとめ等） |
| `{{target_pain}}` | string | ✓ | ターゲットの悩み |
| `{{target_ideal}}` | string | ✓ | ターゲットの理想状態 |
| `{{story}}` | string | | ユーザーのストーリー・体験談 |
| `{{unique_info}}` | string | | 独自情報・専門知識 |
| `{{cta}}` | string | ✓ | CTA（行動喚起） |
| `{{word_count}}` | number | ✓ | 目標文字数（1000/2000/3000/4000） |
| `{{product_link_0}}` ～ `{{product_link_4}}` | string | | アフィリエイトリンク挿入位置（最大5件） |

## 出力形式仕様

Gemini API のレスポンスは `parseContent()` で以下のフォーマットとしてパースされる:

```
# [記事タイトル]
[記事本文（Markdown）]
```

プロンプトでは必ずこの出力形式を指示すること。

## maxTokens 設定

文字数に応じて動的に `maxTokens` を設定する（`RealVertexAiClient` 内で計算）:

| wordCount | maxTokens |
|---|---|
| 4000字 | 12000 |
| 3000字 | 9000 |
| 2000字 | 6000 |
| 1000字以下 | 4000 |

## 変更履歴

| 日付 | 変更内容 | 変更理由 | 効果 |
|---|---|---|---|
| （プロジェクト開始） | 初版作成 | 記事生成機能の実装 | - |

> 変更を加えた際はこの表に追記すること。「効果」は実際に生成した記事を確認してから記入する。

## 品質基準

生成記事が以下を満たしているかを確認する:

- [ ] タイトルがターゲットの悩みを反映している
- [ ] 文字数が指定の ±10% 以内
- [ ] `{{product_link_N}}` が正しく URL に置換されている
- [ ] Markdown の見出し構造が適切（H2/H3 を使用）
- [ ] CTA が記事末尾に自然に組み込まれている

## A/Bテスト方針

プロンプトの大幅な改善を試みる場合:

1. 現行版を `content-generation-v1.md` にコピー
2. 新版を `content-generation-v2.md` として作成
3. `PromptLoader` でどちらを使うか環境変数で切り替え（`PROMPT_VERSION=v2`）
4. 同一テーマで両バージョンの記事を生成して比較
5. 品質が改善されたら `content-generation.md` に昇格させ、古いバージョンを削除
